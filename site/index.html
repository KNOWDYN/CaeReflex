Problem first

CAE evidence is scattered.

A simulation folder can hide useful facts in mesh files, solver dictionaries, result files, scripts, notes, and literature searches. CaeReflex turns that evidence into a clear record that humans and AI agents can inspect safely.

  • ▧Engineers need traceability. Know which files were inspected and what was only inferred.
  • ▧Researchers need context. Connect cases with CrossRef metadata without claiming full-paper reading or validation.
  • ▧Agents need guardrails. Provide warnings, provenance, and do-not-claim rules before summaries are trusted.

Provenance

03

Start locally in minutes.

No hosted backend is required for the homepage or the package workflow. Use a local checkout, then export the evidence you need.

Three-command path

Install, inspect a case, and serve REST/OpenAPI actions when an agent needs tools.

pip install -e ".[server]"
caereflex inspect examples/openfoam_cavity_minimal \
  --out caereflex.json \
  --agent-context agent_context.json \
  --report case_report.md
caereflex serve --host 127.0.0.1 --port 8765
04

REST/OpenAPI for agent workflows.

Browser-hosted agents can call a tunneled local server, import cases, retrieve context, inspect flags, and export reports.

GET  /health
GET  /openapi.yaml
POST /cases/import
GET  /cases/{case_id}/agent-context
GET  /cases/{case_id}/inspection-flags
POST /cases/{case_id}/crossref/attach

Agent-safe sequence

Health check → import case → retrieve agent context → list inspection flags → summarize only detected evidence → ask a qualified human to review engineering conclusions.

05

Clear safety boundary.

This is a trust feature. CaeReflex helps inspection and documentation, but it does not replace engineering validation.

Do not claim that CaeReflex:

  • runs solvers;
  • validates simulations;
  • proves convergence;
  • assesses mesh adequacy;
  • certifies engineering results;
  • approves design safety;
  • reads full papers from CrossRef metadata;
  • replaces qualified engineering judgement.
