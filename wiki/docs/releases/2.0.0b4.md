# 2.0.0b4 — Gate 7 deterministic physics rules

CaeReflex `2.0.0b4` introduces the versioned `caereflex.physics-rule/1.0` protocol and the first built-in rule pack, `openfoam.cfd-core/1.0.0`. ReflexCase schema remains `1.0`, and the backend-neutral inspection contract remains `2.0-alpha.3`.

## Delivered

- six explicit rule-evaluation states: consistent, inconsistent, unknown, not applicable, not evaluated and blocked;
- stable applicability, evidence requirement, remediation and limitation contracts;
- exact JSON-pointer evidence references;
- deterministic run IDs and canonical SHA-256 reports;
- bounded ArrayRef verification through the existing content-addressed registry;
- OpenFOAM mesh, patch, field, dimensional, boundary-reference and time-control checks;
- `caereflex rules` CLI commands;
- a dedicated Python 3.10–3.12 Gate 7 workflow.

## Interpretation boundary

CaeReflex does not validate simulations. A consistent rule result is not proof of convergence, mesh adequacy, turbulence-model suitability, physical validity, experimental validation, certification or design safety.
