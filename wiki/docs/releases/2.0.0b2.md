# CaeReflex 2.0.0b2

`2.0.0b2` implements Gate 6B by mapping validated native OpenFOAM, Gmsh and VTK evidence into the canonical Gate 6A spatial graph.

## Version alignment

- Package: `2.0.0b2`
- ReflexCase schema: `1.0`
- Inspection contract: `2.0-alpha.3`
- Spatial graph format: `1.0`
- Spatial mapping format: `caereflex.spatial-mapping/1.0`

## Native mapping

- OpenFOAM cases map to dataset, aggregate node, face, cell and explicit patch entities.
- Gmsh files map geometry entities, mesh aggregates and physical groups without collapsing geometry into mesh identity.
- VTK files map datasets, point and cell aggregates, fields and collection-reference inventories.
- Heavy coordinates, connectivity, membership and fields remain content-addressed `ArrayRef` links.

## Coordinate-frame posture

OpenFOAM and Gmsh frames remain unresolved unless explicit frame evidence exists. VTK origin and direction can establish an explicit frame, but no length unit is inferred. The mapper does not assume metres, zero origin, global axes or right-handed orientation.

## Integration

Deep and forensic native inspection persists the graph in the shared SQLite state store and attaches only compact references and mapping reports to ReflexCase. Mapping failures preserve the native result and emit stable diagnostics.

## Safety boundary

CaeReflex does not validate simulations, prove geometry equivalence, infer adjacency, assess mesh adequacy, establish physical validity, certify an engineering result or establish design safety.

Gate 6C remains responsible for bounded spatial queries and cross-format acceptance testing. Cross-format equivalence is not asserted in this release.
