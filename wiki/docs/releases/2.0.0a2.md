# CaeReflex 2.0.0a2

`2.0.0a2` adds the first native deep-inspection backend on top of the Gate 5A runtime. ReflexCase schema remains `1.0`, and the backend-neutral contract remains `2.0-alpha.3` because no required serialized contract changed.

## Included

- Isolated `openfoam.native` execution backend.
- Bounded ASCII decoding for OpenFOAM points, faces, owner, neighbour and boundary files.
- Mesh counts, patch ranges and coordinate bounds.
- Time-directory inventory and per-time field metadata.
- Lazy array handles for supported mesh and field data.
- Scalar, vector and common tensor internal-field support.
- Ordered parser-attempt and fallback records.
- Gate 4 dimension vectors preserved in native field summaries.

## Safety boundary

The reader does not run OpenFOAM, load solver libraries, expand includes, execute coded constructs or alter source files. Binary files and unsupported grammar produce diagnostics and metadata-only fallbacks.

The worker remains defence in depth rather than a complete operating-system sandbox. Native inspection does not validate simulations, prove convergence, assess mesh adequacy, certify engineering results or establish design safety.

## Compatibility

Core installation remains dependency-light and does not require OpenFOAM. Existing standard-profile OpenFOAM inspection remains available. Native decoding is selected only for deep and forensic OpenFOAM profiles or through the explicit execution backend.
