# CaeReflex 2.0.0a2

`2.0.0a2` adds the first production native reader to the Gate 5A execution foundation. ReflexCase schema remains `1.0` and the backend-neutral contract remains `2.0-alpha.3`.

## Included

- Bounded `openfoam.native` subprocess backend.
- Native OpenFOAM ASCII point, face, owner, neighbour and boundary decoding.
- Exact mesh counts, patch ranges and axis-aligned point bounds.
- Lazy coordinate, connectivity, owner and neighbour arrays.
- Scalar, vector and tensor field metadata.
- Uniform and nonuniform internal-field arrays.
- Numeric time-directory inventory and field availability by time.
- Gate 4 quantity semantics for native fields and dimensioned properties.
- Literal-only detection of includes, substitutions, code streams, coded conditions and dynamic libraries.
- Explicit binary detection and structured fallback.
- Native parser-attempt ledgers, partial-success results and a complete offline cavity fixture.

## Compatibility

Incomplete OpenFOAM cases continue to use `core.manifest-audit`. Existing schema-v1 ReflexCase payloads remain valid. Native evidence is added through existing metadata, asset, result-field, diagnostic, quantity-evidence and `ArrayRef` collections.

## Security and safety

The reader does not run OpenFOAM, expand dictionaries, invoke shell commands or mutate source files. Binary payloads are not guessed. Includes, substitutions and executable constructs remain literal and produce diagnostics.

CaeReflex does not validate simulations, prove convergence, assess mesh adequacy, certify engineering results or establish design safety.

## Deferred

- binary OpenFOAM decoding;
- decomposed processor reconstruction;
- mesh-quality derivation;
- spatial evidence graphs;
- physics-consistency rules;
- Gmsh and VTK native readers.
