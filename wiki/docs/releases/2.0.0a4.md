# CaeReflex 2.0.0a4

`2.0.0a4` completes Gate 5 native-reader coverage by adding bounded VTK inspection on top of the safe execution runtime. ReflexCase schema remains `1.0`, and the backend-neutral contract remains `2.0-alpha.3` because serialised additions remain optional and backward-compatible.

## Included

- Isolated `vtk.native` execution backend.
- PyVista/VTK-first, meshio-second, core-reader-third fallback order.
- Dependency-free legacy ASCII VTK decoding.
- Dependency-free single-piece XML VTK decoding for inline ASCII and uncompressed inline base64 arrays.
- Points, bounds, structured extents, rectilinear axes, connectivity, offsets and cell-type summaries.
- Point, cell and field data behind content-addressed lazy arrays.
- `.pvd`, multiblock and parallel metadata inventories.
- Time-value discovery without hidden reference loading.
- Stable diagnostics for read, native-reader, encoding, core-reader and collection-reference fallbacks.
- Dedicated optional-backend CI for real meshio and PyVista/VTK paths.

## Reader honesty

A file may be detected but not decoded. A collection may be inventoried while referenced datasets remain unselected or unavailable. Appended, compressed and binary encodings require an optional reader; otherwise CaeReflex records fingerprint-only evidence and information loss.

Coordinate and field units are unresolved unless explicit source evidence or a reviewed annotation supplies them. Cell dimensions are topological classifications, not proof of a physically valid model. CaeReflex does not validate simulations, prove convergence, assess mesh adequacy, certify results or establish design safety.

## Security boundary

The VTK backend does not launch ParaView, execute pipelines, fetch external references, mutate sources or expose complete arrays in ReflexCase JSON. Optional PyVista/VTK libraries execute native code inside a defence-in-depth worker, not a complete operating-system sandbox.

## Gate status

Gate 5 is complete at the native-reader level:

- Gate 5A — safe execution, artefacts and lazy arrays;
- Gate 5B — native OpenFOAM inspection;
- Gate 5C — native Gmsh inspection;
- Gate 5D — native VTK inspection.

Spatial evidence graphs, coordinate-frame reconciliation, physics-consistency rules and project lifecycle services remain deferred to Gates 6–8.
