# 2.0.0b5 — Gate 8 lifecycle services

Release token: `2.0.0b5`

Gate 8 adds project, revision and run lifecycles around CaeReflex evidence, together with deterministic temporal comparisons, immutable human-review records, bounded asynchronous jobs and a bounded REST surface.

## Protocols

- `caereflex.lifecycle/1.0`
- `caereflex.temporal-comparison/1.0`
- `caereflex.human-review/1.0`
- `caereflex.async-job/1.0`

## Main additions

- Projects with active and archived states.
- Immutable, SHA-256-verified ReflexCase revision snapshots.
- Restricted run state transitions and append-only run events.
- Deterministic JSON-pointer temporal comparisons with bounded detail and comparison digests.
- Append-only human reviews with supersession links, optional signature metadata and digest chaining.
- Bounded local asynchronous inspection and comparison jobs using the persistent job store.
- Workspace-bound REST paths, bounded request bodies, bounded lists, bounded metadata and fixed worker/queue capacity.
- A new `caereflex lifecycle` CLI command group and a Python 3.10–3.12 Gate 8 acceptance workflow.

## Compatibility

ReflexCase schema remains `1.0`. The backend-neutral inspection contract remains `2.0-alpha.3`. Existing synchronous case, CrossRef and export endpoints remain available.

## Boundaries

CaeReflex records human review but does not authenticate reviewer identity or authority. Temporal differences do not imply physical significance. Asynchronous jobs are local and not restart-resumable. Gate 8 does not validate simulations, certify designs or replace qualified engineering judgement.
