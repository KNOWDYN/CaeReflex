# CaeReflex 2.0.0a1

`2.0.0a1` is the first public alpha of the CaeReflex 2.x execution foundation. ReflexCase schema version remains `1.0`; the backend-neutral contract version is `2.0-alpha.3`.

## Included

- Gates 1–3 contracts, CLI compatibility and bounded discovery.
- Gate 4 Pint-backed dimensions, units and conservative quantity semantics.
- Gate 5A bounded subprocess execution, parser-attempt records and persistent jobs.
- Immutable SHA-256 local artefacts.
- Additive `ArrayRef` v2 metadata.
- Bounded lazy-array describe, sample, slice and reduction commands.
- Deep-profile execution through the safe `core.manifest-audit` backend.

## Not included

This alpha does not yet provide native OpenFOAM polyMesh reading, native Gmsh geometry/topology, native VTK/PyVista reading, spatial graphs, physics rule packs, distributed execution or autonomous engineering decisions.

## Compatibility

Existing schema-v1 ReflexCase payloads remain valid because all new fields are optional or stored in existing additive metadata collections. Plugins should declare their supported contract range and must not assume that a native execution backend is installed.

## Security and safety

The worker process provides bounded execution and Python-level guards; it is not a complete operating-system sandbox. CaeReflex still does not run solvers, validate simulations, prove convergence, assess mesh adequacy, certify engineering results or establish design safety.

Before depending on this alpha in a persistent workflow, preserve the original CAE source, review diagnostics and retain a copy of the generated manifest and execution ledger.
