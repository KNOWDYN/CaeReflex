# 2.0.0b6 — Platform positioning and release coherence

`2.0.0b6` consolidates the public identity and release metadata of the current CaeReflex 2.0 beta platform after completion of the native inspection, spatial evidence, deterministic physics-rule and governed lifecycle gates.

## Main additions

- Repositioned the repository and GitHub Pages site around deterministic engineering-evidence infrastructure for Physics-AI.
- Organised the platform narrative around four integrated layers: Inspect, Structure, Reason and Govern.
- Made the current native-reader, evidence-contract, spatial-query, physics-rule and lifecycle architecture visible to technical and strategic evaluators.
- Added restrained commercial and strategic evaluation pathways without naming prospective buyers or asserting unsupported adoption, valuation or performance.
- Aligned package, citation, security and wiki release metadata with the current beta line.
- Corrected stale wiki navigation and release facts that predated Gates 7 and 8.

## Compatibility

This release does not change runtime, CLI, REST or OpenAPI behaviour from `2.0.0b5`. ReflexCase schema remains `1.0`. The backend-neutral inspection contract remains `2.0-alpha.3`, and all Gate 5–8 protocols retain their existing independent versions.

Existing native OpenFOAM, Gmsh and VTK inspection, spatial graphs and queries, deterministic physics-rule reports, lifecycle records, temporal comparisons, human reviews and bounded jobs remain compatible.

## Release posture

`2.0.0b6` is a consolidation release: it makes the implemented platform and its ownership, licensing, safety and evaluation pathways coherent across the package and public documentation. It does not introduce a new solver, validation engine or autonomous engineering authority.

## Boundaries

CaeReflex does not validate simulations, prove convergence, establish mesh adequacy, certify engineering results, establish regulatory compliance or determine design safety. A deterministic rule result only describes consistency with the recorded evidence and the applicable versioned rule.
