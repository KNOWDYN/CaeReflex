# CaeReflex 2.0.0a3

`2.0.0a3` adds native Gmsh inspection on top of the Gate 5A execution runtime. ReflexCase schema remains `1.0`, and the backend-neutral contract remains `2.0-alpha.3` because the serialised contract additions remain optional and backward-compatible.

## Included

- Isolated `gmsh.native` execution backend.
- Optional meshio-first reading for supported `.msh` files.
- Dependency-free bounded ASCII support for Gmsh MSH 2.x and 4.x.
- Node tags, coordinates, element tags/types, entity membership, physical membership and ragged connectivity behind lazy arrays.
- Physical names, entity records, mesh dimension, bounds and cell-type summaries.
- Gmsh `NodeData` and `ElementData` fields.
- meshio point and cell data when the optional dependency is installed.
- Safe `.geo` declaration parsing for literal points, curves, loops, surfaces, volumes and physical groups.
- Default fingerprint-only handling for STEP, IGES and BREP files.
- Explicit opt-in Gmsh API entity inspection without mesh generation.
- Ordered meshio, core ASCII and fingerprint fallback records.

## Safety boundary

CaeReflex does not execute `.geo` scripts. Includes, loops, conditionals, system calls, functions, extrusions and boolean operations are preserved as unsupported or unresolved declarations rather than evaluated.

The optional Gmsh API path is disabled by default, never receives `.geo` files, and does not request mesh generation. The worker remains defence in depth rather than a complete operating-system sandbox.

CaeReflex does not validate simulations, validate geometry, assess mesh quality, establish coordinate units, prove convergence, certify engineering results or establish design safety.

## Compatibility

Core installation remains dependency-light. The built-in ASCII and declaration readers require no Gmsh or meshio installation. Existing standard-profile Gmsh inspection remains available. Native decoding is selected for deep and forensic Gmsh profiles or through the explicit execution backend.
