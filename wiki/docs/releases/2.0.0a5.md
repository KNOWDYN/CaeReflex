# CaeReflex 2.0.0a5

`2.0.0a5` completes the Gate 5 acceptance programme by freezing one cross-backend execution envelope and adding deterministic malformed-input coverage across OpenFOAM, Gmsh and VTK.

## Version alignment

- Package: `2.0.0a5`
- ReflexCase schema: `1.0`
- Backend-neutral contract: `2.0-alpha.3`
- Gate 5 backend envelope: `caereflex.gate5.backend-result/1.0`

The new envelope is additive. Backend-specific evidence remains available under the existing execution summary.

## Frozen built-in backends

The following built-ins now expose the same `summary.gate5_compatibility` field set:

- `core.manifest-audit`
- `openfoam.native`
- `gmsh.native`
- `vtk.native`

The common report records backend and plugin identity, inspection profile, read-only posture, source-execution posture, evidence status, relative-path posture and array, artefact, diagnostic and parser-attempt counts.

## Worker-side validation

Before a backend result is persisted, the worker rejects:

- non-object payloads or payloads without a summary object;
- non-JSON and non-finite values;
- absolute or traversal-bearing source paths;
- inconsistent array or diagnostic counts;
- ArrayRef and artefact handles outside the SHA-256 content-addressed store;
- ArrayRef backend identities that disagree with the executing backend;
- large materialised numerical sequences that should be represented by ArrayRef;
- excessive nesting or sequence size.

A violation produces `CRX-GATE5-COMPAT-001` and a failed execution result without terminating the parent process.

## Cross-backend acceptance

The compatibility harness verifies identical envelope fields, deterministic strict JSON, relative paths, content-addressed heavy arrays, parser-attempt accounting and source immutability across all four frozen built-ins.

## Malformed-input coverage

Deterministic offline fixtures cover:

- OpenFOAM count mismatches, binary mesh payloads, negative labels and unclosed face lists;
- Gmsh binary headers, truncated node sections, duplicate node tags and unclosed sections;
- VTK binary legacy payloads, truncated point records, invalid connectivity, malformed XML and appended XML;
- invalid backend payloads including NaN, unsafe paths, missing summaries, heavy raw arrays and mismatched counts.

Reader-level failures degrade to explicit fallback evidence. Backend-contract violations fail before persistence. In both cases, source bytes remain unchanged.

## Safety boundary

This release does not execute solvers, meshers, ParaView, scripts or shell commands. It does not fetch collection references, infer units, repair malformed files, assess mesh quality, prove convergence or certify design safety. CaeReflex does not validate simulations.

The default worker remains defence in depth rather than a complete operating-system sandbox. Native libraries and hostile files may require stronger external isolation.

## Gate status

Gate 5 is complete across:

- Gate 5A — safe execution, artefacts and lazy arrays;
- Gate 5B — native OpenFOAM inspection;
- Gate 5C — native Gmsh inspection;
- Gate 5D — native VTK inspection;
- Gate 5 completion — frozen cross-backend compatibility and malformed-input acceptance.

Canonical spatial evidence, coordinate-frame reconciliation, physics-consistency rule packs and project lifecycle services remain deferred to Gates 6–8.
