# REST API

CaeReflex exposes a FastAPI REST service for importing CAE cases into the configured workspace, storing `ReflexCase` records, attaching CrossRef literature evidence, and exporting case artifacts for agent workflows.

The implementation lives in `caereflex/server/app.py`; this document describes the routes and wire formats exposed by that app.

## Startup

Install the server extra before starting the API:

```bash
pip install -e ".[server]"
```

Start the server with an explicit workspace:

```bash
caereflex serve --host 127.0.0.1 --port 8765 --workspace /path/to/workspace
```

The command starts Uvicorn and prints the OpenAPI URL. The workspace is the root used for persisted cases under `.caereflex/cases/` and for path-oriented operations.

## Security model

- **Localhost mode:** `--host 127.0.0.1` or `--host localhost` can run without `--api-key`.
- **Non-localhost mode:** any other host, such as `0.0.0.0`, requires `--api-key`; the CLI exits before starting the server if the key is missing.
- **Authenticated calls:** when an API key is configured, callers send it with the `x-api-key` HTTP header.

Example authenticated request:

```bash
curl -H "x-api-key: $CAEREFLEX_API_KEY" \
  http://127.0.0.1:8765/cases
```

The server enforces the API key for externally bound hosts. If the key is missing or wrong, protected routes return `401`.

## Workspace path model

`POST /cases/import` accepts a `path` in the request body.

- Relative paths are resolved against the configured workspace.
- Absolute paths are resolved as absolute filesystem paths.
- When the server is externally bound, `assert_safe_workspace_path` validates the resolved import path and prevents path traversal outside the configured workspace.

Recommended agent behavior is to send workspace-relative paths, for example:

```json
{
  "path": "lid_driven_cavity",
  "adapter": "auto"
}
```

Avoid sending `../` segments or absolute paths for remote/agent-facing deployments.

## OpenAPI discovery

FastAPI exposes the JSON schema automatically, and CaeReflex also exposes a YAML representation:

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/openapi.json` | OpenAPI schema generated by FastAPI. |
| `GET` | `/openapi.yaml` | Same OpenAPI schema serialized as YAML. |

Example:

```bash
curl http://127.0.0.1:8765/openapi.json
curl http://127.0.0.1:8765/openapi.yaml
```

## Request models

### `ImportRequest`

Used by `POST /cases/import`.

```json
{
  "path": "openfoam_cavity_minimal",
  "adapter": "auto",
  "attach_crossref": false,
  "return_agent_context": true,
  "options": {}
}
```

Fields:

| Field | Type | Default | Notes |
| --- | --- | --- | --- |
| `path` | string | required | Workspace-relative path is recommended. |
| `adapter` | string | `auto` | `auto`, `gmsh`, `openfoam`, `vtk`, or adapter aliases supported by the service. |
| `attach_crossref` | boolean | `false` | If true, attempts to attach CrossRef literature evidence during import. |
| `return_agent_context` | boolean | `true` | If true, response includes `data.agent_context`. |
| `options` | object | `{}` | Reserved for caller metadata/options; currently not consumed by `app.py`. |

### `CrossRefRequest`

Used by `POST /cases/{case_id}/crossref/search` and `POST /cases/{case_id}/crossref/attach`.

```json
{
  "query": "lid driven cavity CFD benchmark",
  "include_case_tags": true,
  "limit": 5,
  "mailto": "researcher@example.com",
  "mock_response": null
}
```

Fields:

| Field | Type | Default | Notes |
| --- | --- | --- | --- |
| `query` | string or null | `null` | Explicit CrossRef query. If omitted, service derives context from the case. |
| `include_case_tags` | boolean | `true` | Adds case tags to the search context when supported. |
| `limit` | integer | `10` | Maximum number of literature records to request. |
| `mailto` | string or null | `null` | Optional CrossRef etiquette contact email. |
| `mock_response` | string or null | `null` | Optional mock payload/path used by tests or offline workflows. |

### `ExportRequest`

Used by markdown and BibTeX export routes. `POST /cases/{case_id}/export/json` accepts the same body shape but returns the case JSON directly instead of writing `out`.

```json
{
  "out": "reports/case_123.md"
}
```

Fields:

| Field | Type | Default | Notes |
| --- | --- | --- | --- |
| `out` | string or null | route-specific filename | Relative output path under the configured workspace. Markdown defaults to `{case_id}.md`; BibTeX defaults to `{case_id}.bib`. |

## Endpoint reference

### `GET /health`

Readiness endpoint. Does not require a request body.

Response example:

```json
{
  "status": "success",
  "service": "caereflex",
  "version": "0.1.0"
}
```

### `GET /version`

Returns the package version.

Response example:

```json
{
  "version": "0.1.0"
}
```

### `POST /cases/import`

Imports and inspects a CAE artifact, stores the resulting case in `.caereflex/cases/`, and optionally returns agent context.

Request example:

```json
{
  "path": "openfoam_cavity_minimal",
  "adapter": "openfoam",
  "attach_crossref": false,
  "return_agent_context": true,
  "options": {}
}
```

Imported case response example:

```json
{
  "status": "success",
  "case_id": "case_openfoam_cavity_minimal_8f3a1c2d",
  "summary": "OpenFOAM case detected with standard system, constant, and initial-condition folders.",
  "warnings": [],
  "inspection_flags": [],
  "provenance_summary": [
    "inspected_path",
    "adapter_selected"
  ],
  "next_recommended_actions": [
    "get_agent_context",
    "export_case_report"
  ],
  "data": {
    "agent_context": {
      "case_id": "case_openfoam_cavity_minimal_8f3a1c2d",
      "case_name": "openfoam_cavity_minimal",
      "summary": "OpenFOAM case detected with standard system, constant, and initial-condition folders.",
      "inspection_flags": [],
      "literature_evidence": []
    }
  }
}
```

### `GET /cases`

Lists stored cases from the configured workspace.

Response example:

```json
{
  "status": "success",
  "cases": [
    {
      "case_id": "case_openfoam_cavity_minimal_8f3a1c2d",
      "case_name": "openfoam_cavity_minimal",
      "case_type": "openfoam",
      "path": "/path/to/workspace/.caereflex/cases/case_openfoam_cavity_minimal_8f3a1c2d.json"
    }
  ]
}
```

### `GET /cases/{case_id}`

Returns the full serialized `ReflexCase` for a stored case.

Response example:

```json
{
  "case_id": "case_openfoam_cavity_minimal_8f3a1c2d",
  "case_name": "openfoam_cavity_minimal",
  "case_type": "openfoam",
  "detected_formats": ["OpenFOAM"],
  "detected_tools": ["OpenFOAM"],
  "inspection": {
    "status": "success"
  },
  "inspection_flags": [],
  "literature_evidence": []
}
```

### `GET /cases/{case_id}/summary`

Returns a compact case summary for dashboards and agent triage.

Summary response example:

```json
{
  "status": "success",
  "case_id": "case_openfoam_cavity_minimal_8f3a1c2d",
  "summary": "OpenFOAM case detected with standard system, constant, and initial-condition folders.",
  "detected_formats": ["OpenFOAM"],
  "detected_tools": ["OpenFOAM"]
}
```

### `GET /cases/{case_id}/agent-context`

Returns an agent-optimized context object. This is the preferred second call after importing a case.

Agent context response example:

```json
{
  "case_id": "case_openfoam_cavity_minimal_8f3a1c2d",
  "case_name": "openfoam_cavity_minimal",
  "summary": "OpenFOAM case detected with standard system, constant, and initial-condition folders.",
  "detected_formats": ["OpenFOAM"],
  "detected_tools": ["OpenFOAM"],
  "inspection_flags": [
    {
      "severity": "warning",
      "category": "mesh_quality",
      "message": "No mesh quality report was found.",
      "path": "constant/polyMesh"
    }
  ],
  "literature_evidence": []
}
```

### `GET /cases/{case_id}/literature`

Returns attached literature records and aggregate literature context.

Literature response example:

```json
{
  "literature_evidence": [
    {
      "title": "A benchmark study for lid-driven cavity flow",
      "doi": "10.1234/example.doi",
      "source": "CrossRef",
      "year": 2024,
      "url": "https://doi.org/10.1234/example.doi"
    }
  ],
  "literature_context": {
    "status": "success",
    "query": "lid driven cavity CFD benchmark",
    "record_count": 1
  }
}
```

### `GET /cases/{case_id}/inspection-flags`

Returns inspection warnings and errors only.

Inspection flags response example:

```json
{
  "inspection_flags": [
    {
      "severity": "warning",
      "category": "missing_file",
      "message": "Expected transportProperties was not found.",
      "path": "constant/transportProperties"
    }
  ]
}
```

### `POST /cases/{case_id}/crossref/search`

Searches CrossRef using a stored case and returns candidate records without modifying the stored case.

Request example:

```json
{
  "query": "OpenFOAM cavity validation",
  "include_case_tags": true,
  "limit": 3,
  "mailto": "researcher@example.com",
  "mock_response": null
}
```

Response example:

```json
{
  "records": [
    {
      "title": "OpenFOAM validation studies for cavity flow",
      "doi": "10.1234/openfoam.validation",
      "source": "CrossRef",
      "year": 2023,
      "url": "https://doi.org/10.1234/openfoam.validation"
    }
  ],
  "literature_context": {
    "status": "success",
    "query": "OpenFOAM cavity validation",
    "record_count": 1
  }
}
```

### `POST /cases/{case_id}/crossref/attach`

Searches CrossRef, attaches returned records/context to the stored case, saves the updated case, and returns the full serialized case.

Request body is `CrossRefRequest`.

Response example:

```json
{
  "case_id": "case_openfoam_cavity_minimal_8f3a1c2d",
  "case_name": "openfoam_cavity_minimal",
  "literature_evidence": [
    {
      "title": "OpenFOAM validation studies for cavity flow",
      "doi": "10.1234/openfoam.validation",
      "source": "CrossRef",
      "year": 2023
    }
  ],
  "literature_context": {
    "status": "success",
    "record_count": 1
  }
}
```

### `POST /cases/{case_id}/export/json`

Returns the full serialized case as JSON. It accepts `ExportRequest` for schema consistency, but `out` is ignored by the current implementation.

Request example:

```json
{
  "out": null
}
```

Response example: same shape as `GET /cases/{case_id}`.

### `POST /cases/{case_id}/export/markdown`

Writes a Markdown report under the configured workspace.

Request example:

```json
{
  "out": "reports/openfoam_cavity_minimal.md"
}
```

Export path response example:

```json
{
  "status": "success",
  "out": "reports/openfoam_cavity_minimal.md"
}
```

### `POST /cases/{case_id}/export/bibtex`

Writes a BibTeX file under the configured workspace.

Request example:

```json
{
  "out": "reports/openfoam_cavity_minimal.bib"
}
```

Export path response example:

```json
{
  "status": "success",
  "out": "reports/openfoam_cavity_minimal.bib"
}
```

## Error behavior examples

### Invalid API key

When the server is bound to a non-localhost host and the `x-api-key` header is missing or incorrect, protected endpoints return `401`.

```json
{
  "detail": "API key required outside localhost."
}
```

### Unsafe path

In externally bound mode, paths resolved outside the configured workspace are rejected by `assert_safe_workspace_path`. `POST /cases/import` catches this as a bad request.

Example request:

```json
{
  "path": "../outside-workspace/case.foam",
  "adapter": "auto"
}
```

Example response:

```json
{
  "detail": "Path is outside configured workspace: /path/to/outside-workspace/case.foam"
}
```

### Unknown case ID

Stored cases are loaded from `.caereflex/cases/{case_id}.json`. Unknown IDs generally surface as file-loading errors.

Example response:

```json
{
  "detail": "Not Found"
}
```

or, depending on the caller and exception handler:

```json
{
  "detail": "[Errno 2] No such file or directory: '/path/to/workspace/.caereflex/cases/case_missing.json'"
}
```

### Unsupported adapter

If `adapter` is not one of the supported adapter names, import fails with a `400` response.

Example request:

```json
{
  "path": "openfoam_cavity_minimal",
  "adapter": "nastran"
}
```

Example response:

```json
{
  "detail": "Unsupported adapter: nastran"
}
```

### Missing optional dependencies

If the server dependencies are not installed, the CLI cannot import FastAPI/Uvicorn and exits with the dependency error path. Install the server extra:

```bash
pip install -e ".[server]"
```

If `create_app()` is called directly without FastAPI installed, it raises:

```text
RuntimeError: FastAPI is not installed. Install caereflex[server].
```

## Agent-specific notes

### Custom GPT Actions

- Use the OpenAPI schema from `/openapi.json` or `/openapi.yaml`.
- If the server is exposed through a tunnel or any non-localhost host, configure the `x-api-key` header in the Action authentication settings.
- Prefer this call sequence:
  1. `POST /cases/import` with a workspace-relative path.
  2. `GET /cases/{case_id}/agent-context` to gather compact reasoning context.
  3. `GET /cases/{case_id}/inspection-flags` to focus follow-up questions on warnings/errors.

### Claude-style tool callers

- Treat the workspace as the safe root and pass relative paths to `/cases/import`.
- Do not infer filesystem access beyond the configured workspace.
- Prefer this tool order:
  1. `/cases/import`
  2. `/cases/{case_id}/agent-context`
  3. `/cases/{case_id}/inspection-flags`
- Use `/cases/{case_id}/literature` or CrossRef endpoints only when literature evidence is needed for the task.
